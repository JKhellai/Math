## 骨架要学会算术

上一节的 ℕ 只会一件事:从 0 出发,一步步走。它不会加、不会乘——「+」这个符号根本尚未定义,1 + 1 = 2 暂时还是一句没有意义的话。可加法的**直觉**早就藏在骨架里:n + m,不就是「从 n 出发再走 m 步」么?这一节把这句话铸成严格的定义,然后做一件更要紧的事:交换律、结合律这些自幼熟习、却从未被证明过的定律,将在这里逐条获得证明。

## 递归:归纳的孪生兄弟

「从 n 出发再走 m 步」是个**分层指示**:第 0 步在哪儿说清楚,每下一步怎么走说清楚。把这种定义方式提炼成一般原理:

**递归定理(recursion theorem).** 设 X 是集合,a ∈ X,g : X → X。则**存在唯一**的函数 f : ℕ → X,使得

f(0) = a,  且  ∀n ∈ ℕ, f(S(n)) = g(f(n))。

读法:f(0) 钦定为 a(起步),f 在「下一个数」处的值由它在「这个数」处的值经 g 加工而来(传递)。这与[[num_n|归纳原理]]是一对孪生:归纳说「性质沿链条传遍全体」,递归说「**取值**沿链条铺满全体」——一个用于证明,一个用于定义。

又是「存在且唯一」,照[[fond_funzioni|∃!]]的老规矩拆两半。**唯一性**我们当场证掉:设 f、f′ 都满足两条,对 P(n) :≡「f(n) = f′(n)」跑归纳——奠基:f(0) = a = f′(0);归纳步:设 f(n) = f′(n),则 f(S(n)) = g(f(n)) = g(f′(n)) = f′(S(n))。由归纳原理,f 与 f′ 逐点相同,而二者定义域、到达域已同,故 f = f′。∎

**存在性**却深得多,值得说破为什么:函数不是一句咒语,是一个**实实在在的集合**(那堆有序对)——归纳法只会证明「已有之物」的性质,**变不出集合来**。要把无穷多步指示真正拼装成一堆合法的有序对,得在集合论里动一番手脚。此处暂且承认存在性,严格搭建留待 [[num_ricorsione|递归定理]] 专节,先继续前行。

## 加法

固定 m ∈ ℕ。对「X = ℕ,起步 a = m,传递 g = S」用一次递归定理,得到唯一的函数 σₘ : ℕ → ℕ,满足 σₘ(0) = m、σₘ(S(n)) = S(σₘ(n))。把 σₘ(n) 写作 **m + n**,上面两条就成了**加法(addition)的定义**:

- (A1) m + 0 := m;
- (A2) m + S(n) := S(m + n)。

读出声来:「加 0 = 原地不动」「加『n 的下一个』= 先加 n、再走一步」——正是「重复地走」的严格版。顺带重提一句旧话:这样一来 + 是个堂堂正正的函数 ℕ×ℕ → ℕ,(m,n) ↦ m+n——「运算就是函数」,当初在函数那节埋下的伏笔,在此第一次落到实处。

**1 + 1 = 2,这就证。** 按定义 1 := S(0),2 := S(1)。于是

1 + 1 = 1 + S(0) ⟨A2⟩= S(1 + 0) ⟨A1⟩= S(1) = 2。∎

两步,每步都有出处。从今天起,这个等式不再是口诀,是定理。

顺手再得一个小定理:**S(n) = n + 1**——因为 n + 1 = n + S(0) ⟨A2⟩= S(n + 0) ⟨A1⟩= S(n)。「后继」从此可以退居幕后,我们终于有资格写熟悉的 n + 1 了。

## 定律是定理,不是定义

现在试试 0 + n。别急着喊「当然等于 n」——定义直接给出的只有 m + **0** = m(A1,0 在**右边**),而 **0** + n 里 0 在左边,定义对它**只字未提**。定义是不对称的,对称性必须证明。这就是本节的主戏:每一条「天经地义」,都要由一次归纳确立为定理。

**引理 A.** ∀n ∈ ℕ, 0 + n = n。

**证明.** 对 n 归纳。奠基:0 + 0 ⟨A1⟩= 0。归纳步:设 0 + n = n,则 0 + S(n) ⟨A2⟩= S(0 + n) ⟨归纳假设⟩= S(n)。∎

**引理 B.** ∀m, n ∈ ℕ, S(m) + n = S(m + n)。(「左边的进位也能搬出来」——A2 只管右边,左边这条得证。)

**证明.** 固定 m,对 n 归纳。奠基:S(m) + 0 ⟨A1⟩= S(m) ⟨A1 反用⟩= S(m + 0)。归纳步:设 S(m) + n = S(m + n),则 S(m) + S(n) ⟨A2⟩= S(S(m) + n) ⟨归纳假设⟩= S(S(m + n)) ⟨A2 反用⟩= S(m + S(n))。∎

**定理(加法交换律 commutativity).** ∀m, n ∈ ℕ, m + n = n + m。

**证明.** 固定 m,对 n 归纳。奠基:m + 0 ⟨A1⟩= m ⟨引理 A⟩= 0 + m。归纳步:设 m + n = n + m,则

m + S(n) ⟨A2⟩= S(m + n) ⟨归纳假设⟩= S(n + m) ⟨引理 B 反用⟩= S(n) + m。∎

留意这场仗的**结构**:交换律不是一锤子买卖,它踩着两块垫脚石(引理 A 管「0 在左边」,引理 B 管「后继在左边」)才够得着。把大目标拆成小引理、逐块归纳——这是往后无数证明的标准兵法。**结合律** (k + m) + n = k + (m + n) 反倒是一场单挑,不需要任何引理,留给你在文末练手;它一到手,括号就可以全扔了,放心写 k + m + n。

## 乘法

同一台递归机器,再次启用。固定 m,定义**乘法(multiplication)**:

- (M1) m · 0 := 0;
- (M2) m · S(n) := m · n + m。

「乘 0 归零」「乘『n 的下一个』= 先乘 n、再加一个 m」——乘法就是**重复地加**,正如加法是重复地走。验收一下,算 2 · 2(其中 3 := S(2),4 := S(3)):

2 · 2 = 2 · S(1) ⟨M2⟩= 2·1 + 2 = (2 · S(0)) + 2 ⟨M2⟩= (2·0 + 2) + 2 ⟨M1⟩= (0 + 2) + 2 ⟨引理 A⟩= 2 + 2,

而 2 + 2 = 2 + S(1) ⟨A2⟩= S(2 + 1) = S(2 + S(0)) ⟨A2⟩= S(S(2 + 0)) ⟨A1⟩= S(S(2)) = S(3) = 4。∎

每一步都有判例可援——机器虽慢,字字有据。

乘法的定律同样逐条可证,且**踩在加法定律的肩上**。示范一条,看这种「层层叠加」:

**定理(分配律 distributivity).** ∀k, m, n ∈ ℕ, k · (m + n) = k·m + k·n。

**证明.** 固定 k、m,对 n 归纳。奠基:k·(m + 0) ⟨A1⟩= k·m ⟨A1 反用⟩= k·m + 0 ⟨M1 反用⟩= k·m + k·0。归纳步:设 k·(m+n) = k·m + k·n,则

k·(m + S(n)) ⟨A2⟩= k·S(m + n) ⟨M2⟩= k·(m+n) + k ⟨归纳假设⟩= (k·m + k·n) + k ⟨加法结合律⟩= k·m + (k·n + k) ⟨M2 反用⟩= k·m + k·S(n)。∎

看见没——倒数第二步用的正是你即将在练习里证明的**加法结合律**:定律之间有先后的依托——加法诸律先行确立,乘法诸律方能在其上立足。乘法的交换律、结合律手法如出一辙(各需一两块「0 在左、S 在左」式的垫脚石),清单先记在这儿,手痒尽可自证,我们不再逐条占篇幅。

## 一道激活练习

证明**加法结合律(associativity)**:

∀k, m, n ∈ ℕ, (k + m) + n = k + (m + n)。

(提示:固定 k、m,对 n 归纳——想想为什么挑 n:定义 A1/A2 都是在**右边**的变元上动作的,对 n 归纳,每一步都有现成的判例可援。全程只需 A1、A2 与归纳假设,连引理都不用。)

## 参考解答

(建议先自己写完再看。)固定 k, m,对 P(n) :≡「(k+m)+n = k+(m+n)」跑归纳。

**奠基** P(0):

(k + m) + 0 ⟨A1⟩= k + m ⟨A1 反用,作用在 m 上⟩= k + (m + 0)。

**归纳步**:设 (k+m)+n = k+(m+n),往证 P(S(n)):

```
(k + m) + S(n)
= S((k + m) + n)      ⟨A2⟩
= S(k + (m + n))      ⟨归纳假设⟩
= k + S(m + n)        ⟨A2 反用⟩
= k + (m + S(n))      ⟨A2 反用,作用在内层⟩
```

两步齐备,由归纳原理结论对一切 n 成立;k、m 任意,定理得证。∎

回味:整场证明就是把括号外的 S「往里搬运」——A2 正着用是「拆」,反着用是「装」,归纳假设负责中途换轨。做完这道,你已亲手确立三大定律之一;随之而来的还有记号上的自由:从此 k + m + n 不必加括号,怎么结合都是同一个数。
