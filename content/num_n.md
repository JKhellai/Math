## 从借用到建立

九节地基,我们一路「暂且借用」自然数:当例子、当下标、当清单的编号。从这一节起,借用结束,严格的建立开始。第一步,是直面一个平时谁也不问的问题:**自然数究竟是什么?**

别急着答「0, 1, 2, 3, …」——那三个点正是问题所在:「……」凭什么继续?往哪儿继续?Peano 在 1889 年的洞见是:把数数这件事剥到只剩骨头,剩下的不过两样——**一个起点,一个「下一个」**。从 0 出发,一步、一步、再一步,永不回头、永不撞车、无一漏网。把这三个「永」写成公理,自然数的全部本质就被捉住了。

## 五条公理

我们约定:存在一个集合 ℕ、它的一个元素 0、以及一个函数 S : ℕ → ℕ(读「**后继**(successor)」,S(n) 直觉上就是 n 的下一个),满足:

- **(P1)** 0 ∈ ℕ。(有起点。)
- **(P2)** S 是 ℕ 到 ℕ 的函数。(人人有下一个,且只有一个——这正是「函数」一词所保证的。)
- **(P3)** S 是[[fond_iniettive_suriettive|单射]]。(不同的数,下一个也不同——步伐永不撞车。)
- **(P4)** 0 ∉ im S。(0 不是任何数的下一个——起点没有前任,路永不绕回开头。)
- **(P5) 归纳原理(principle of induction)**:若 M ⊆ ℕ 满足 0 ∈ M,且 ∀n (n ∈ M ⟹ S(n) ∈ M),则 M = ℕ。(从 0 出发步步可达的,就是全部——没有「天外来客」。)

先停下来欣赏一件事:这五条公理**一个新原始概念都没引进**。「函数」「单射」「像」「子集」——全是地基里既有的概念,Peano 公理不过是用我们已经千锤百炼的语言写下的五句话。地基九节备下的语言,在此第一次整体派上用场。

> 两句立场话。其一,记号:我们取 0 为起点(有的传统从 1 起,纯属约定;0 起步在集合论与计算里更顺手)。其二,身份:这五条是**公理**——我们宣布「凡满足这五条的 (ℕ, 0, S) 就叫自然数」,而不去追问 ℕ「真正」由什么做成。在更深的集合论里,确实能用纯集合亲手造出一个满足五条的模型(冯·诺依曼:0 := ∅, 1 := {∅}, 2 := {∅,{∅}}, …),且任何两个满足五条的系统本质上是同一个——这两件事都留待将来;眼下采取公理式立场,这是一个选择,也是 Amann–Escher 的走法。

- **三元环** {0, 1, 2},S 依次 0↦1↦2↦0。验一验:有起点(P1✓)、S 是函数(P2✓)、S 单射(三个元素去向各不同,P3✓)、归纳也过(含 0 又步步封闭的子集只能是全体,P5✓)。可 S(2) = 0,即 0 ∈ im S——**P4 排除它**:路不许绕回起点。少了 P4,「数数」可以是个轮回。
- **打结的尾巴** {0, 1, 2},S 为 0↦1, 1↦2, 2↦2。P1✓;P2✓;P4✓(im S = {1,2},不含 0);P5✓(同上)。可 S(1) = S(2) = 2 而 1 ≠ 2——**P3 排除它**:步伐不许撞车。少了 P3,前进可以原地踏步、历史可以坍缩。
- **天外来客**:正经的 ℕ 链,旁边再飘一条两头无限的链(……↦ a ↦ b ↦ ……,与 ℕ 不相交,内部 S 单射、0 也不在像里)。P1–P4 全过。可取 M = 正经链那一半:0 ∈ M、步步封闭,M 却不等于全体——**P5 排除它**:不许有从 0 出发到达不了的元素。归纳原理的真身,就是「**步步可达即全部**」。

三个反例各被一条公理排除——这正是反例的作用:把「为什么每条都不能去掉」钉死。

## 归纳:从公理到武器

P5 说的是子集;实战里我们更常对**性质**做归纳。两者一纸之隔:

**定理(归纳证明原理).** 设 P(n) 是关于自然数的[[fond_quantificatori|谓词]]。若 (i) P(0) 真(**奠基**),且 (ii) ∀n ( P(n) ⟹ P(S(n)) )(**归纳步**),则 ∀n ∈ ℕ, P(n)。

**证明.** 令 M := { n ∈ ℕ : P(n) }——描述法,从 ℕ 里切,合法。(i) 说 0 ∈ M;(ii) 说 n ∈ M ⟹ S(n) ∈ M。由 P5,M = ℕ,即 P 对一切自然数成立。∎

证明两行,分量千钧:这是我们手里第一件能**一口气搞定无穷多个命题**的武器。此前要断言「∀n …」,要么靠定义直接展开,要么束手;现在只需两步——立住 0,接好每一环。往后整个数系构造、乃至分析里无数定理,都骑在这两步上。

用法的要点也在这两步里:奠基别忘验(多少错误「证明」死在第 0 块多米诺没立稳);归纳步里 P(n) 是**假设**(叫归纳假设),你的任务是从它推出 P(S(n))——不是重新证 P(n),也不是直接证 P(S(n))。

## 一道激活练习

用归纳原理证明:**没有数是自己的后继**,即

∀n ∈ ℕ, S(n) ≠ n。

(提示:奠基处想想哪条公理直接管「S(0) 与 0」;归纳步是个「假设 ≠、推出 ≠」的局面——正面硬推不顺手时,回想第一节的逆否/反证,再想想 P3 给了你什么本事。)

## 参考解答

(建议先自己写完再看。)思路:对 P(n) :≡「S(n) ≠ n」跑归纳;奠基靠 P4,归纳步靠 P3,恰好一人一岗。

**奠基** P(0):要证 S(0) ≠ 0。S(0) 当然在 im S 里,而 P4 说 0 ∉ im S——若 S(0) = 0,则 0 ∈ im S,矛盾。故 S(0) ≠ 0。

**归纳步**:设 P(n) 成立,即 S(n) ≠ n(归纳假设),往证 P(S(n)),即 S(S(n)) ≠ S(n)。用反证:假设 S(S(n)) = S(n)。这是一个「S(□) = S(△)」形状的等式,P3(S 单射)的本事正是剥壳:得 S(n) = n。这与归纳假设矛盾。故 S(S(n)) ≠ S(n)。

两步齐备,由归纳原理,∀n ∈ ℕ, S(n) ≠ n。∎

回味:P4 守住起点、P3 守住每一步——再看一眼上面三张赝品图,「打结的尾巴」里 2 正是自己的后继,它骗过了 P4 与 P5,恰恰死在本定理用的 P3 手里。公理与反例,在这道题里完成了一次相互印证。

## 往前看一步

此刻的 ℕ 还只是一副骨架:能从 0 出发一步步走,仅此而已——它还不会加、不会乘,连 1 + 1 = 2 都无从谈起,因为「+」根本尚未定义。下一步是给这副骨架配上运算,而思路已经藏在它自身的结构里:加法无非是「**重复地走**」——n + m 就是从 n 出发再走 m 步。把这句话变成严格的定义,需要一种新的定义方式(顺着 S 一层层往上铺),它正是归纳原理在「定义」一侧的孪生兄弟。这就是 [[num_aritmetica|自然数的算术]] 要做的事。
