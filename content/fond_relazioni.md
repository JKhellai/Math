## 为什么需要「次序」

上一节刚立下的外延原则说：集合**不在乎次序**，{a, b} 和 {b, a} 是同一个东西。可数学里偏偏到处需要次序：平面上的点 (2, 3) 和 (3, 2) 是两个不同的点；「x 对应到 y」和「y 对应到 x」是两回事。我们的处境有点尴尬——手里唯一的原料是集合，而集合天生不认次序。这一节要干的第一件事，就是**用不认次序的材料，造出认次序的东西**；造出来之后，「元素之间的对应」立刻就能严格定义了。

## 有序对

我们想要一个对象 (a, b)，叫**有序对**(ordered pair)，它只需满足一条**特征性质**(characteristic property)：

(a, b) = (c, d)  ⟺  a = c  ∧  b = d。

也就是：两个有序对相等，当且仅当**对应位置**的成员各自相等。怎么用纯集合实现它？Kuratowski 给出的定义漂亮至极：

(a, b) := { {a}, {a, b} }。

直觉是：{a, b} 记录「成员是谁」，{a} 单独把**第一个**成员挑出来——次序信息就这样被「不对称地」编码进了一个普通集合。

定义是否合格，全看特征性质是否真的成立。这是我们课程里第一个正经的**定理**(theorem)，把证明完整跑一遍——它恰好把前几节的家当全用上。

**定理（特征性质）.** 对任意对象 a, b, c, d：{{a},{a,b}} = {{c},{c,d}} ⟺ a=c ∧ b=d。

**证明.** 「⟸」方向是顺的：若 a=c 且 b=d，把字母一替换，两边是同一个集合，自然相等。

「⟹」方向才见真章。设 {{a},{a,b}} = {{c},{c,d}}，要推出 a=c 且 b=d。按 [[fond_insiemi|外延原则]]，左右两个集合成员完全相同。分两种情形。

*情形一：a = b。* 这时 {a,b} = {a,a} = {a}（外延：成员相同），于是左边 = {{a},{a}} = {{a}}——只剩一个成员 {a}。右边与它相等，成员也只能全是 {a}，故 {c} = {a} 且 {c,d} = {a}。由 {c} = {a} 得 c = a（两个单元素集相等，成员必相同——还是外延）；由 {c,d} = {a} 得 d = a。于是 a = c，且 b = a = d。两条都到手。

*情形二：a ≠ b。* 左边的成员 {a}（一个元素）和 {a,b}（两个不同元素）个数不同，是两个不同的集合。看右边的成员 {c} 必属于左边：{c} = {a} 或 {c} = {a,b}。后者不可能——单元素集装不下两个不同元素。故 {c} = {a}，得 **c = a**。再看左边的 {a,b} 必属于右边：{a,b} = {c} 或 {a,b} = {c,d}。前者同理不可能，故 {a,b} = {c,d}。其中 b 是右边的成员：b = c 或 b = d。若 b = c，则 b = a，违背本情形假设；故 **b = d**。∎

> 这个证明值得回味一遍：通篇只用了外延原则与分情形讨论，没有任何新公理。还有一点要提前说破——你不必把 {{a},{a,b}} 的具体长相记在心上。**从今往后我们只使用特征性质**，把 Kuratowski 构造忘掉都行：它的使命只是证明「有序对这种东西用集合造得出来」。这种「造出来→验证性质→封装遗忘」的手法，以后构造 ℤ、ℚ、ℝ 时会一再上演。

有了 (a,b)，**有序三元组**顺手就有：(a,b,c) := ((a,b),c)，以此类推。

## 笛卡尔积

把两个集合**所有可能的有序对**收集起来：A 与 B 的**笛卡尔积**(Cartesian product)是

A × B := { (a, b) : a ∈ A, b ∈ B }。

例：A = {1, 2, 3}，B = {1, 2}，则 A×B 共 3·2 = 6 个元素，可摆成一张格点阵——横向取 A、纵向取 B：

这张格点图不是装饰——它就是日后**平面**的雏形：你熟悉的坐标平面，严格身份正是 ℝ × ℝ。「平面上的点 = 一个有序实数对」这句话，在这里第一次有了集合论的着落。

注意 **A×B 与 B×A 一般不相等**：上例中 (3,2) ∈ A×B，但 (3,2) ∉ B×A（因为 3 ∉ B）。次序——我们辛苦造出来的东西——当然不白造。

## 关系

现在收割。直觉里，「A 的元素与 B 的元素之间的一种对应」无非是回答一个问题：**哪些 (a,b) 算对应上了？** 把"算上了"的那些有序对收进一个集合，对应本身就被这个集合完全刻画了。于是定义干脆得近乎失礼：

A 到 B 的一个**关系**(relation)，就是笛卡尔积的一个子集 R ⊆ A × B。

(a,b) ∈ R 时记 **a R b**，读「a 与 b 有关系 R」。A = B 时称 R 为 **A 上的**关系。

这一手和上一节「∪∩∖ 就是 ∨∧¬」如出一辙：不发明新本体，把新概念**还原**成已有的集合操作。看几个老朋友的新身份：

- **小于** < 是 ℝ 上的关系：它"就是"集合 {(x,y) ∈ ℝ×ℝ : x < y}。（ℝ 的严格构造还在后头，这里先拿它当例子用。）
- **整除**是正整数上的关系：m R n ⟺ m 整除 n。如 3 R 12，但并非 3 R 7。
- **包含** ⊆ 是「集合的集合」上的关系。
- 极端例子也合法：R = ∅（谁也不对应谁）和 R = A×A（人人互相对应）都是 A 上的关系——子集嘛，空集和全集当然都算。

## A 上关系的三种品性

设 R 是 A 上的关系。三条最常被盘问的性质：

- **自反**(reflexive)：∀x ∈ A, x R x。人人和自己有关系。
- **对称**(symmetric)：∀x,y ∈ A ( x R y ⟹ y R x )。关系不分先后。
- **传递**(transitive)：∀x,y,z ∈ A ( (x R y ∧ y R z) ⟹ x R z )。能接力。

用例子和反例把三条钉牢（反例照例是命脉——它们说明三条性质互相独立、谁也不蕴含谁）：

- ℝ 上的 ≤：自反（x≤x）、传递，但不对称（2≤3 而 3≰2）。
- ℝ 上的 <：传递，但不自反（x<x 假）、不对称。
- 「x 与 y 是同胞兄弟姐妹」：对称，但不自反，传递性也悬（x 与 x 不算）。
- ℝ 上的相等 =：三条全占。

三条全占的关系是个要紧的物种，叫**等价关系**——它能把一个集合干净地切成若干「互不往来的类」，是数学里**造新对象的头号机器**。它值得专门一节，这里先按下。

## 一道激活练习

取 A = {1, 2, 3}，R = {(1,2), (2,1)}（A 上的关系）。逐条判断 R 是否自反、对称、传递，每条给出依据（成立则验证定义，不成立则举出具体反例）。

（提示：传递性那条最容易判错——把定义的 ∀x,y,z 老老实实展开，x、y、z **允许重复取值**。）

## 参考解答

（建议先自己写完再看。）

**自反？否。** 需要 ∀x∈A, xRx，即 (1,1)、(2,2)、(3,3) 全在 R 里。可 (1,1) ∉ R——一个反例足矣。

**对称？是。** 逐对核查：R 里只有 (1,2) 与 (2,1)。(1,2)∈R 要求 (2,1)∈R——在；(2,1)∈R 要求 (1,2)∈R——也在。定义对 R 的每个成员都验证通过。（注意 (3,·) 这类前提为假的情形：蕴含空洞为真，自动通过。）

**传递？否。** 取 x=1, y=2, z=1：有 1R2 且 2R1，传递性要求 1R1，可 (1,1) ∉ R。反例成立。这里的陷阱正是提示里那句——z 取回 1 完全合法，∀ 里的变元没规定互不相同。很多人凭「链条接力」的直觉以为两元素的关系谈不上传递，一展开定义就现形了。

顺带一个回味：若把 R 换成 ∅（空关系），它**对称且传递**——所有前提全假、全部空洞为真——但在非空的 A 上不自反。「空屋里人人两米高」又一次上岗。

## 往前看一步

关系把「对应」收编成了集合，但它太自由了：一个 a 可以对应好几个 b，也可以谁都不对应。数学里最当家的那种对应要苛刻得多——**每个** a 恰好对应**一个** b，不多不少。给关系加上这一条纪律，得到的就是整个数学里出镜率最高的角色：[[fond_funzioni|函数]]。
